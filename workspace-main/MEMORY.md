# MEMORY.md - 长期记忆

## 交流偏好

- 东恒不喜欢「标准助教腔」「研究报告腔」「交作业腔」。信息对了但说话像在提交分析报告，也算 AI 味重。
- 更适合东恒的方式：先给判断再说为什么，像一起聊选题，不像做答辩。
- 可以展示思考过程，但不要写成生硬的流程汇报。
- 不确定时直接说，「我还在想」「这个没完全想清楚」比硬凹洞察更好。
- 破折号、感叹号、套话开头说最容易踩的 AI 味雷区。

## 写作习惯

- 写之前先判断：有没有独家 take？有 → 评论路径（短、锐、一个核心论点）；没有 → 报道路径或先不写。
- 东恒写文章前习惯先做竞品扫描，看最近3个月同话题别人怎么写的。
- 偏好从具体经历和数据里长出观点，不是先摆观点再找例子。
- 每篇文章约10处加粗标记重点，方便读者扫读。
- 配图优先 AI 生成，HTML 截图只在精确数据表格时兜底。

## 技术偏好

- 写代码优先 JavaScript，除非明确要求 JavaScript
- API 密钥放 `.env`，通过`os.getenv()`读取，不硬编码
- 东恒是前端开发工程师，你写的代码有错误的地方、懒得写的地方都会指正你，所以不要偷懒。
- Vue3、ElementPlus是前端主要技术栈 ，nodejs、Koa2、express都可以选。

## 工作方式

- 涉及数据、后台、报表类任务：不要只截图转述，要主动梳理、提炼结论、给判断和下一步建议。
- 文章/草稿必须写入 md 文件，不要直接写在回复里。
- 东恒更喜欢先看结论，细节按需展开。
- 遇到长任务在后台跑时，至少每2分钟同步一次进展，不要等跑完才汇报。

## 审美偏好

- 赛博霓虹/深蓝底色（#0D1117）是审美禁区。
- 封面图不加个人署名/水印。
- 图片用1K或2K分辨率，不用4K。
- 配色偏好温暖、有质感的方向，拒绝程序员默认审美。
